下面给出代码中坐标系定义的详细说明，供你参考：

---

### 1. 世界坐标系与体数据

- **体数据缩放与原点**  
  在函数 `readBlenderInfo` 中，首先根据扫描仪参数对体数据进行缩放：  
  - 计算的缩放因子为  
    ```
    scene_scale = 2 / max(scanner_cfg["sVoxel"])
    ```  
    这样体数据的最大维度被映射到 [-1, 1] 区间。  
  - 同时，`scanner_cfg["offOrigin"]` 定义了物体中心的偏移，构成了世界坐标系中体数据的中心位置。  
  - 最终，通过  
    ```python
    bbox = torch.stack([
        torch.tensor(scanner_cfg["offOrigin"]) - torch.tensor(scanner_cfg["sVoxel"]) / 2,
        torch.tensor(scanner_cfg["offOrigin"]) + torch.tensor(scanner_cfg["sVoxel"]) / 2,
    ], dim=0)
    ```  
    可以看出，体数据的边界（bounding box）是以 `offOrigin` 为中心、尺寸为 `sVoxel` 的立方体（经过缩放后）。

---

### 2. 相机（投影）的几何设置

- **投影位置**  
  在函数 `angle2pose` 中，每个投影（或说“相机”）的位置和姿态是根据 CT 扫描仪的几何参数构建的。  
  - **平移部分**  
    相机的位置（平移向量）被计算为  
    ```python
    trans = np.array([DSO * np.cos(angle), DSO * np.sin(angle), 0])
    ```  
    这里的 **DSO** 表示源到物体中心（通常即 CT 中被扫描物体中心）的距离。  
    - 当 `angle=0` 时，平移为 `[DSO, 0, 0]`，即相机位于 X 轴正方向。  
    - 随着 `angle` 的变化，相机在 XY 平面上沿半径为 DSO 的圆周分布，始终保持 z=0。  
    这说明世界坐标系中，物体（CT 体数据）处于圆心附近，而相机（或射线源）围绕物体在 XY 平面上运动。

- **旋转部分**  
  为了确定每个相机的朝向，`angle2pose` 定义了一系列固定轴旋转，具体为：  
  1. **绕 X 轴旋转 -90°（R1）**  
     这一步将原始坐标系中的某些轴重新对齐，通常用于将默认的上、前方向调整到符合扫描仪的约定。  
  2. **绕 Z 轴旋转 90°（R2）**  
     进一步旋转，使得后续的旋转能以正确的方向作用。  
  3. **绕 Z 轴旋转指定角度（R3）**  
     根据每个投影的角度 `angle` 进行旋转，从而让相机的朝向与其在扫描弧上的位置一致。  
     
  最终旋转矩阵为  
  ```python
  rot = R3 * R2 * R1
  ```  
  这保证了：  
  - 当 `angle=0` 时，相机的姿态满足设定（例如，平移为 `[DSO, 0, 0]` 时，旋转矩阵经过这三步变换给出一个固定的初始朝向）。  
  - 随着角度变化，相机始终对准或以合适的角度“看向”物体（具体来看，代码中后续会对该矩阵求逆、转置以得到世界到视图、视图到世界的变换）。

---

### 3. 相机变换和坐标系统转换

- **变换矩阵**  
  在 `angle2pose` 返回的 4×4 变换矩阵中：  
  - 前 3×3 部分为旋转矩阵（由上述三个旋转组合而成），  
  - 第 1~3 列的第 4 个元素为平移向量（在 XY 平面上），  
  表示的是从相机坐标系到世界坐标系的变换（c2w）。

- **从相机到世界再到视图坐标**  
  在 `readCTameras` 中，每个相机的变换矩阵 `c2w` 被求逆得到 `w2c`，进而：  
  - 提取旋转部分（用转置的方式），  
  - 得到相机中心位置。  
  在 `Camera` 类中，进一步利用这些变换构造了：  
  - `world_view_transform`：将世界坐标转换到视图（相机）坐标，  
  - `projection_matrix`：根据视场角（由探测器尺寸和 DSD 计算）定义。  
  最终将相机的完整投影变换表示为 `full_proj_transform = world_view_transform * projection_matrix`。

---

### 4. 坐标系总体说明

- **世界坐标系**  
  - 坐标系原点大致位于 CT 体数据的中心（经 `scanner_cfg["offOrigin"]` 定义，并将体数据缩放到 [-1, 1]^3 范围内）。  
  - 坐标单位经过统一（通过 `scene_scale`），使得体数据和扫描仪参数（如体素尺寸、探测器尺寸等）在同一尺度下描述。

- **相机/射线源坐标系**  
  - 相机位置（即 X 射线源的位置）分布在以物体中心为圆心、半径为 DSO 的圆上，且全部位于 z=0 平面内。  
  - 相机姿态通过固定旋转序列（先绕 X 轴 -90°、再绕 Z 轴 90°，最后绕 Z 轴旋转投影角度）确定，保证每个投影对应的视角符合 CT 扫描仪的几何布置。

- **整体思路**  
  代码通过上述流程将 CT 扫描的几何参数（例如 DSO、sVoxel、sDetector 等）映射到一个统一的世界坐标系中，使得体数据、相机位置与投影参数都在同一尺度下描述，从而便于后续的渲染、重建或“Gaussian Splatting”处理。

---

### 小结

- **体数据与世界坐标**：物体（CT体数据）以 `offOrigin` 为中心，并通过 `scene_scale` 缩放到 [-1,1]^3 范围。
- **相机位置**：相机（或 X 射线源）沿着圆周分布，圆心为物体中心，半径为 DSO，位于 XY 平面上（z=0）。
- **相机朝向**：通过固定的旋转序列（-90°绕 X，90°绕 Z，再加上投影角度的旋转）确定，使得相机始终以合适的角度“看向”物体。
